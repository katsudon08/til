- useEffectは第二引数が変わったタイミングで、第一引数のfnを走らせる
- useEffectの実行は前回のcleanUp -> 今回の本体の順番で走る

すなわち以下のような場合

```ts
useEffect(() => {
  const a = b;
  a.start();
  return (() => a.destroy())
}, [...])
```

第二引数が変わって実行されるたびに、壊して、作って始めるを何回も繰り返すことになり、しかもそれは新しい別物として作られるので重い。

特にReactできにしなくちゃいけないのはレンダリングのたびに関数が再生成されてしまうこと。
だからこそ、再生成されないように気をつけたり、逆に適切に再レンダリングを走らせるなどを行う必要がある。
